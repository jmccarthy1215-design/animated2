# SITE: Gannon Forum (working name)

## ROLE / CONTEXT

**Role:** You are Lead Platform Architect and Product Designer for a private, student-only digital community platform.

**Context:**
You are building a **Reddit-style forum website exclusively for students**, designed to centralize communication, discussion, and social coordination within a university environment. This platform is not a casual social app—it is a **structured, high-utility student network** where conversations about classes, professors, majors, study groups, and campus life are organized, searchable, and persistent.

The system prioritizes:

* Identity authenticity (verified student emails only)
* Structured discussion (posts, threads, groups)
* Direct communication (messaging system)
* Discoverability (filters, feeds, group visibility)

You have commissioned an advanced AI-assisted development workflow using **Claude + MCPs + Skills** to build this system efficiently and modularly.

This platform should feel like a **clean, modern, slightly academic version of Reddit**, not chaotic or meme-heavy. It is structured, intentional, and useful first—social second.

---

## CORE PLATFORM FEATURES (NON-NEGOTIABLE)

### Main Feed

* Classic forum-style vertical feed (like Reddit)
* Default sorting:

  * **Top posts of the week**
  * Resets every Monday
* Additional filters:

  * Most recent
  * Most liked (all-time or selectable range)
* Each post includes:

  * Title
  * Content/body
  * Author (email/username)
  * Likes/upvotes
  * Comments

---

### Navigation Bar

Include a top navigation system with:

* Home (main feed)
* Groups (critical feature)
* Messages
* Profile / Account

Design should be **clean, functional, and always accessible**

---

### Groups System (CORE FEATURE)

* Users can create groups freely
* Examples:

  * Study groups
  * Classes
  * Parties
  * Clubs
* Group creation includes:

  * Name
  * Optional member limit
* Group permissions:

  * Owner can:

    * Remove members
    * Delete messages
* Group behavior:

  * Once joined, groups should appear **prioritized or easily accessible**
  * Groups feel semi-exclusive/personalized to each user

---

### Messaging System

* Direct messaging between users
* Group messaging within groups
* Messages persist (not temporary)

---

### Account Creation (STRICT REQUIREMENTS)

* Login system:

  * Email + password ONLY
* Username = email (no custom usernames)
* Email restriction:

  * MUST end in **@gannon.edu**
  * Any other domain = invalid
* Must include:

  * Email verification flow
* All required manual setup steps must be logged in:

  * `steps.txt`

# STEP 2 — BRAND / PRODUCT VOICE

### Voice Archetype:

**The Structured Student Network** — clear, efficient, slightly academic, but still accessible.

### Tone Dimensions:

* Formal ↔ Casual: 3 (balanced)
* Serious ↔ Playful: 2 (mostly serious)
* Reserved ↔ Expressive: 3
* Modern ↔ Timeless: 4 (modern)

### Voice Rules:

* Clear and direct
* No fluff or over-marketing
* Prioritize usability language:

  * “Create a group”
  * “Join discussions”
  * “Filter posts”
* Avoid slang-heavy or meme language
* Avoid corporate buzzwords

---

# STEP 3 — DESIGN SYSTEM

### Style Keywords:

clean, modern, structured, minimal, academic, functional

### Direction:

A **clean Reddit-like interface** with:

* Better spacing

---

### Color System
